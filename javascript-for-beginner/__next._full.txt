<h3>Data Types</h3>
<p>In other programming languages like <code>C#</code> and <code>C++</code>, there is a most important concept of <strong>Call by Value</strong> and <strong>Call by Reference</strong>.</p>
<p>In the very top level data types are categorized based on how a data value will be stored in memory, and how they can be accessed by the user.</p>
<p>Here we have two category - <code>Primitive</code> ( Call by value ) and <code>Reference</code> | Non-primitive ( Call by reference )</p>
<p><strong>Call by value</strong>: A copy of original value to be passed.
<strong>Call by reference</strong>: Original value to be passed.</p>
<p><strong>Primitive data types</strong></p>
<ul>
<li><code>number</code> ( 198, 143, 1.45 ), Range 2^53 - 1</li>
<li><code>bigint</code> ( 198n ), trading, stock market, reddit like websites</li>
<li><code>string</code> ( "vikash", 'kumar' )</li>
<li><code>boolean</code> ( <code>true</code> / <code>false</code> )</li>
<li><code>null</code> ( standalone value | dataType = <code>Object</code> )</li>
<li><code>undefined</code> ( dataType )</li>
<li><code>symbol</code> ( unique )</li>
</ul>

<h3>Conversions</h3>
<p>It is possible to convert data types in JavaScript.
eg. <code>number</code> -> <code>string</code> | <code>string</code> -> <code>number</code> | <code>number</code> -> <code>boolean</code> | <code>string</code> -> <code>boolean</code> | <code>boolean</code> -> <code>string</code> | <code>boolean</code> -> <code>number</code></p>
<p><strong>Why we need conversion</strong>?
Most of the time browser returns a string value to user.
Whenever we fetch some data from an API. ( eg. Weather API )</p>
<p>Conversion will not become a problem if you know what will be the expected result after conversion.</p>

<h4>PostFix and PreFix operators</h4>
<p>PostFix and PreFix operators work on single value.</p>
<pre><code class="hljs language-js"><span class="hljs-keyword">let</span> gameCounter = <span class="hljs-number">10</span>;

<p>Here you are thinking that both the operators did the same work but they works differently.</p>
<p>| Value: gameCounter = 10 | Current value | Passed value |
| :---------------------- | :------------ | :----------- |
| gameCounter             | 10            | 10           |
| ++gameCounter           | 11            | 11           |
| gameCounter++           | 11            | 12           |
| ++gameCounter           | 13            | 13           |
| gameCounter--           | 13            | 12           |
| --gameCounter           | 11            | 11           |</p>
<blockquote>
<p>We print only current values in the console</p>
</blockquote>
